# Cross-Agent Note: MACE-OFF24 Crambin D1 Gate Result

## Finding

MACE-OFF24 (mace-torch 0.3.15) successfully runs stable NVT dynamics on crambin
(1CRN, 642 atoms with hydrogens) via the OpenMM-ML integration. Vacuum NVT at
300 K with 1 fs timestep and Langevin thermostat produced a stable trajectory
through 10 ps (confirmed) and is continuing through 100 ps (in progress as of
this writing). Energy and temperature are well-behaved with no NaN or extreme
forces detected.

**D1 gate assessment: CONDITIONAL PASS** (10 ps confirmed stable; 100 ps expected
stable within the current SLURM job wall time).

**Critical platform finding:** OpenMM's CUDA platform is INCOMPATIBLE with both
H200 (sm_90) and B200 (sm_100) GPUs due to PTX version mismatch (error code 222:
CUDA_ERROR_UNSUPPORTED_PTX_VERSION). The simulation runs on OpenCL as a fallback,
achieving 1.51 ns/day for 642-atom vacuum crambin. This is approximately 2x slower
than CUDA would be. PyTorch's CUDA works fine on H200 -- the incompatibility is
specifically in OpenMM's compiled CUDA kernels.

**Key technical details:**
- Model: MACE-OFF24-medium (downloaded to `~/.cache/mace/MACE-OFF24_medium.model`)
- ML potential name: `mace-off24-medium` (NOT `maceoff` as some docs suggest)
- OpenMM 8.5.1, openmmml (openmm-ml), PyTorch 2.5.1+cu121
- Platform: OpenCL (CUDA failed, OpenCL succeeded as fallback)
- Speed: 1.51 ns/day (OpenCL, H200, 642 atoms vacuum)
- PE after minimization: -281,976 kJ/mol
- PE during NVT (10 ps): -279,640 +/- 70 kJ/mol
- Temperature: 282-310 K (equilibrating to 300 K target)
- No NaN in positions, forces, or energies through 10 ps

---

## Impact

- **Alpha-M:** MACE-OFF24 is viable for the benchmark but performance is degraded
  on current GPU hardware. For production runs (Phase 2), either:
  (a) Use RTX 5000 Ada nodes (sm_89, which cu121 supports -- CUDA should work there),
  (b) Accept OpenCL performance penalty (~2x slower), or
  (c) Rebuild OpenMM with a newer CUDA toolkit that supports sm_90/sm_100.
  
  At 1.51 ns/day on OpenCL for 642-atom vacuum crambin, a 1 ns production run
  would take ~15.9 hours. Solvated systems will be significantly slower.

- **Combined:** The CUDA platform incompatibility does NOT affect the feasibility
  of the Alpha-M benchmark -- the simulation works correctly, just slower. This
  is a performance issue, not a correctness issue. The D1 gate criterion
  (">= 100 ps stable NVT") is expected to be met.

---

## Recommended Action

1. **For Subphase 1.2 MACE pilot runs:** Test MACE on RTX 5000 Ada nodes
   (`--partition=gpu --gres=gpu:rtx_5000_ada:1`) to verify CUDA works there.
   This could double performance.

2. **For Phase 2 production runs:** If RTX 5000 Ada CUDA works, use those nodes
   for MACE. If not, budget 2x wall time for OpenCL runs compared to CUDA
   estimates in the Implementation Plan.

3. **OpenMM rebuild (low priority):** Rebuilding OpenMM from source with CUDA
   toolkit 12.4+ would fix the PTX issue on H200/B200. This is a significant
   effort and should only be attempted if RTX 5000 Ada CUDA also fails.

4. **SLURM scripting:** Always include platform fallback code (CUDA -> OpenCL -> CPU)
   in MACE simulation scripts. Example pattern is in
   `phases/phase-1/subphase-1.1/output/scripts/mace_crambin_nvt.py`.
