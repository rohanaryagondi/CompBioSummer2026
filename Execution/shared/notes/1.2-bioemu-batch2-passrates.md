# Cross-Agent Note: BioEmu Batch 2 Plan, Oversampling, and Disorder Pre-Screen

**Status:** STUB (v1). Finalized pass-rate comparison pending completion of
7-10 day batch 2 generation. This v1 documents the plan + upstream decisions
so downstream subphases can act on the full picture immediately.

## Finding

Batch 2 has been planned and the first SLURM array submitted (2026-04-19).
Key changes from the Sub 1.1 batch-1 protocol:

1. **Disorder pre-screen upstream.** 10 of 103 candidates were excluded
   BEFORE generation using a sequence-only heuristic (TOP-IDP scale per
   Campen 2008 + FoldIndex per Prilusky 2005). Calibrated against batch-1
   controls: YAP1 (0.7% BioEmu pass) and SYUA (57% pass) both flagged;
   CALM1 (98%), ubiquitin (98%), crambin all pass. Sub 1.1 wasted ~10,368
   NPZ files on YAP1 for 0.7% pass rate; this is exactly the loss avoided.

2. **Class-based per-protein num_samples.** Every protein gets its own
   `num_samples` set by the oversampling formula
   `num_samples = ceil(2000 / pass_rate * 1.3)` with class-assigned
   pass rates:

   | Class | Pass rate | num_samples | Batch 2 count |
   |-------|:--------:|:-----------:|:--:|
   | structured_globular | 0.92 | 2,827 | 53 |
   | idp_like | 0.47 | 5,532 | 21 |
   | large_globular_long | 0.60 | 4,334 | 19 |
   | transmembrane | 0.70 | 3,715 | 0 |
   | multi_domain_metastable | 0.23 | 11,305 | 0 |

   Total planned denoised samples: 348,349 (vs ~95K in batch 1).

3. **RTX 5000 Ada exclusive.** All 93 per-protein SLURM jobs are forced to
   `--partition=gpu --gres=gpu:rtx_5000_ada:1`. A runtime `nvidia-smi` check
   in `bioemu_batch2.sbatch` aborts the job with exit code 99 if a non-RTX
   5000 Ada GPU is allocated (defensive per Sub 1.1 SU-save lesson).

4. **Memory bump for all proteins.** `--mem=40G` is the default (vs 24G in
   batch 1 early jobs), applying the Sub 1.1 SPG1 lesson (high-num_samples
   proteins blew memory at 24G).

## Disorder-Screen Heuristic Note

IUPred3 and ESMfold are not available in env-bioemu on this HPC cluster,
and installing them would violate the non-destructive env-management rule
(operational-practices.md). The sequence-only TOP-IDP + FoldIndex heuristic
in `disorder_screen.py` is a lightweight approximation that correctly
flagged the two known batch-1 failures (YAP1, SYUA) in post-hoc testing.

**Limitation:** this is NOT a published-grade disorder predictor. For any
manuscript-level claim about disorder fraction, a proper IUPred3 or
AlphaFold pLDDT re-screen should be run (Phase 2 action).

## CD19_HUMAN Exclusion (2026-04-26)

**Decision: EXCLUDE.** CD19_HUMAN (P15391, 556 aa) formally excluded from batch 2.

**Root cause:** CD19 is a type I single-pass transmembrane protein with a 253-aa disordered cytoplasmic tail (55.3% disorder-promoting residues). BioEmu generates the full sequence without membrane context, so the disordered tail produces broken backbone bonds that fail physicality filters. Actual pass rate 1.4% (90/6500) — same pathology as YAP1_HUMAN (0.7%, dropped batch 1).

**Why the screen missed it:** CD19's whole-sequence `disorder_fraction=0.5018` falls below the 0.60 exclusion threshold because the structured extracellular Ig-like domains (47% of protein) dilute the signal from the disordered cytoplasmic tail (45% of protein). This is a known blind spot of whole-sequence averaging for mixed-architecture proteins.

**Impact:** 93 → 92 proteins. Still exceeds ≥90 success criterion. Binding assay coverage drops from 2 to 1 in batch 2 (GCN4 remains); combined with batch 1's 6, total is sufficient. Scientifically, CD19 is a second documented case (after YAP1) of BioEmu failure on large disordered regions — a valuable finding for characterizing generative model limitations.

**At-risk proteins monitored:** PABP_YEAST (577 aa, disorder_fraction=0.40) is the next-highest risk but lacks CD19's TM+disorder pathology. All other L>500 proteins are structured (disorder_fraction 0.15-0.31). No additional exclusions expected.

**No replacement needed.** No disorder threshold change needed (tightening would over-exclude).

## Per-protein topup policy (standing — user directive 2026-04-19T17:15Z)

**Any protein that completes with `status=partial`** (bioemu validation
`conformation_count < 0.9 * num_samples_requested`) triggers an automatic
topup by head-1.2. No user approval needed. Rationale: user directive
"never accept partial data. Always go back and get full data, assuming
beneficial." For any partial result, head-1.2 does:

1. Compute actual pass rate = `conformation_count / (943 * bs)` (or using
   existing batches × batch_size).
2. Update the row in `output/scripts/batch2_manifest.csv` in place:
   - `expected_pass_rate` → actual pass rate
   - `num_samples` → `ceil(2827 / actual_pass_rate)` rounded up for margin
     (target: at least 2827 physical conformations = original plan)
3. Submit one-off sbatch with `--array=<idx>` via `bioemu_batch2.sbatch`.
   bioemu auto-resumes from existing batch_*.npz files via its
   `count_samples_in_output_dir` path; only the delta batches are
   generated.
4. Record in this table.

Note: bioemu's `status` field will still read "partial" after topup
(since pass rate < 90% makes the check structurally unreachable). That
is expected; the relevant QC is the absolute `conformation_count`.
