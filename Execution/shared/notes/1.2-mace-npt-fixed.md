## Finding

After 3 rounds of patching investigation (8+ NPT diagnostics across 2 sessions) we
have identified and fixed the MACE-OFF24 + OpenMM NPT instability. The combined
`sentinel-bond + protein HBonds constraints + dt=1 fs` recipe yields stable NPT
trajectories. **Sub 1.4 production should use NPT, NOT the NVT fallback.**

## Production fix recipe

After `MLPotential('mace-off24-medium').createMixedSystem(...)` returns the hybrid
system, apply two patches BEFORE constructing the Simulation:

1. **`add_protein_sentinel_bonds(hybrid_system, topology, protein_atom_indices)`**
   — adds zero-strength `HarmonicBondForce` bonds along the protein bonded graph.
   This restores the connectivity that openmm-ml's `_removeBonds(allInSet=True)`
   stripped, so OpenMM's `findMolecules()` correctly sees the protein as ONE
   molecule (not 534 singletons). Without this, `MonteCarloBarostat::scaleCoordinates`
   and `enforcePeriodicBox` operate per-atom on the protein, tearing it apart.
   Reference: openmm-ml issue #91 (jchodera, 2025-02-04, still open).

2. **`add_protein_hbonds_constraints(hybrid_system, topology, protein_atom_indices)`**
   — re-adds AMBER14 HBonds constraints (heavy-atom-to-H rigid bonds) for the
   protein region by building a parallel reference system and copying the
   constraint distances. Without this, protein C-H/N-H/O-H bonds are unconstrained,
   and the C-H stretching mode (~3000 cm⁻¹, period ~10 fs) is at the edge of
   integrator stability at dt=1 fs. Once `MonteCarloBarostat` perturbs positions,
   the unconstrained-H instability runs away.

Both helpers live in `output/scripts/npt_diagnostics/npt_diag_common.py`.

## Root cause analysis

Two distinct failure modes were operating concurrently:

1. **Singleton-molecule failure** (Rounds 1-2): `findMolecules()` returned
   ~7000 molecules (all protein atoms as singletons), so OpenMM's per-molecule
   re-imaging at integration time wrapped each protein atom independently.
   Bonded atoms ended up on opposite sides of the box → MACE neighbor list
   computed pathological edges → NaN within ~3-25 ps.

2. **Unconstrained-H instability** (Round 3): Even after fixing #1 with sentinel
   bonds, the protein C-H/N-H/O-H bonds remained unconstrained because the
   classical AMBER14 system was built with `constraints=None`. At dt=1 fs, the
   ~10 fs C-H stretching period is on the edge of velocity-Verlet stability.
   `MonteCarloBarostat`-driven box rescales add small perturbations that
   compound with the H atom instability, leading to runaway integration
   (test_J probe data showed protein span exploding from 28 Å to 113 Å in
   ~14 callbacks immediately before NaN at 12.5 ps NPT).

Both failures had to be fixed together — fixing only one extends the failure
horizon but doesn't prevent it.
