# SO3LR Crambin NVT Result (D1 Gate Evidence)

## Summary

SO3LR v0.1.0 successfully completed a 100 ps NVT simulation of crambin (640 atoms,
vacuum, 300 K) on an RTX 5000 Ada GPU. D1 gate minimum criterion is met. A 1 ns
continuation is in progress.

## Key Findings for Alpha-M Track

1. **D1 verdict: PASS.** SO3LR installs and runs stable NVT trajectories on crambin.
   100 ps completed without NaN or instability.

2. **Use the CLI, not the programmatic JAX-MD interface.** The `so3lr nvt` CLI handles
   neighbor lists, thermostat, JIT compilation, and checkpoints internally. A prior
   attempt using custom JAX-MD code failed due to shape broadcasting bugs. The CLI
   is the recommended approach for all future Alpha-M SO3LR simulations.

3. **Geometry relaxation is essential.** Always use `--relax` before MD. The raw PDB
   structure needs minimization (converges in ~140 steps) to avoid NaN in initial MD
   steps.

4. **Performance:** 2.93 ms/step for 640 atoms on RTX 5000 Ada (float32). Extrapolated
   throughput: ~15 ns/day for crambin. Expected to be faster on H200/B200 GPUs.

5. **Vacuum only.** SO3LR does not support implicit or explicit solvent for non-periodic
   systems. All Alpha-M SO3LR simulations will be vacuum or require a periodic box with
   minimum dimension >= 24 A (2 x lr_cutoff).

6. **float32 precision is sufficient** for stable trajectories of 100+ ps. Hamiltonian
   drift is ~15 meV/ps, acceptable for NVT with Nose-Hoover thermostat.

7. **SLURM note:** numpy must be explicitly available in the conda env on compute nodes.
   The env-so3lr environment has been updated with numpy in its site-packages. The SLURM
   script also sets `PYTHONPATH` to include user site-packages as a safety measure.

## What This Means for Subphase 1.2+

- SO3LR can proceed to larger protein pilots (e.g., Tier 1 proteins from Alpha-M proposal)
- CLI command pattern for production: `so3lr nvt --input <file>.xyz --output <traj>.hdf5 --dt 0.5 --temperature 300 --md-cycles N --md-steps 1000 --relax --precision float32 --lr-cutoff 12.0 --restart-save <checkpoint>.npz`
- For restart/continuation: add `--restart-load <prev_checkpoint>.npz --no-relax`
- Memory requirement: 64 GB RAM recommended (JIT compilation is memory-intensive)
- GPU recommendation: Any CUDA-capable GPU; RTX 5000 Ada tested, H200/B200 expected to be faster
