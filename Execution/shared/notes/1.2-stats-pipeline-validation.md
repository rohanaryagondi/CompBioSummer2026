# Cross-Agent Note: Statistical Pipeline Validated and Phase-2-Ready

## Finding

All 5 load-bearing statistical components from `ImplementationPlan.md §12`
and Appendix A are implemented, unit-tested, and cross-validated. The
pipeline is ready for Sub 1.4 pilot analysis (D2 G2/G3/G6 evidence) and
Phase 2 production analysis (combined-paper BF, Alpha-M convergence ICC,
Delta WMSE bootstrap CIs).

**Pipeline location:**
`phases/phase-1/subphase-1.2/output/scripts/stats/`

**Components (IP reference → module):**
- §12.1 Alpha-M primary test (Friedman + Nemenyi) → `friedman_nemenyi.py`
- §12.1 + §10.3 ICC convergence (ICC(2,k), ICC(2,1), attenuation correction,
  block-split attenuation estimator) → `icc.py`
- §12.1 clustering / §14.1 "per-residue N inflation" pre-emption (2-level
  hierarchical bootstrap, 10K iterations) → `hierarchical_bootstrap.py`
- §12.3 + App. A combined-paper integration BF (Fisher-1915 exact closed
  form + 4-prior sensitivity + R subprocess fallback) → `jzs_bf.py`
- §12.1 trajectory truncation (T_min per protein + JSON audit log +
  mdtraj truncate) → `truncation.py`

**Validation outcome:** `python tests/test_all.py` → **exit 0, 28 s total**.

Headline cross-check numbers:
- Python JZS BF == R `BayesFactor::correlationBF` BF to **0.0001 relative
  difference** on n=20 r=0.7605 synthetic data (both return 288.2478).
  The task spec allowed 20%; agreement is effectively machine-precision
  because both tools use the same Fisher closed form with the same
  (rho+1)/2 ~ Beta(sqrt(2), sqrt(2)) prior.
- Hierarchical bootstrap CI is **2.7x wider than naive 1-level bootstrap**
  CI on the same synthetic data, quantifying the reviewer-attack-preemption
  value of hierarchical clustering (IP §14.1).
- ICC convergence-correction recovers **exactly** the asymptotic ICC=0.85
  from measured 0.68 given the 20% attenuation factor (IP §10.3 midpoint).
- Coverage sweep: hierarchical-bootstrap 95% CI covers the population
  truth in **19/20 runs** (nominal 95%; 2K iterations; n=14 proteins).

## Impact

- **alpha-m:** Sub 1.4 can directly invoke `friedman_nemenyi`, `icc_2k`, and
  `hierarchical_bootstrap` on real pilot S2 R^2 tables to score D2 gates
  G2 (Friedman across generators) and G3 (ICC convergence). `truncation.py`
  supplies the T_min truncation step required before any Alpha-M per-protein
  generator comparison. Block-split attenuation estimator
  (`icc.estimate_attenuation_from_blocks`) replaces the 0.20 midpoint
  default with a per-pilot empirical factor in Sub 1.4.

- **gamma:** `hierarchical_bootstrap` is the CI primitive for Gamma's
  Wilcoxon effect-size intervals (IP §12.2). No separate gamma-specific
  code is needed.

- **delta:** `hierarchical_bootstrap` is the CI primitive for Delta's
  WMSE → Spearman top-k-DEG evaluation. FDR (BH/BY) and calibration live
  in task-005's scope; `jzs_bf` / `icc` are not used by Delta.

- **combined:** The JZS BF pipeline IS the combined-paper integration
  primary test. The 4-prior sensitivity (`four_prior_sensitivity`) produces
  the table required by IP §12.3 decision rule (BF_10 > 3 under JZS AND
  BF_10 > 1 under skeptical). For paper-grade BF values (August
  manuscript), call `jzs_correlation_bf_r` so reviewers can point at
  `BayesFactor::correlationBF` directly -- Python and R agree to
  machine precision so the paper number does not change.

- **infrastructure / operations:** New conda env `env-stats` was created
  per the non-destructive-env rule. No existing env was modified. R + R
  packages (BayesFactor, jsonlite) ship inside `env-stats` so operators
  do not need to maintain a separate R installation. Env-stats also has
  mdtraj and pymc, so the env is self-contained for the full pipeline.
