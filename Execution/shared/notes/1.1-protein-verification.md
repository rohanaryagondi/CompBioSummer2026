# Cross-Agent Note: 12-Protein Benchmark Verification Audit

## Executive Summary

After a full desk audit of the 12 remaining Alpha-M proteins, the benchmark is
**mostly airtight, with zero new RED-level risks**, but carries **six YELLOW
risks** that warrant pre-Phase-2 flags. No protein should be dropped as a result
of this audit, which means T5 still holds at 12 and the combined-paper viability
remains intact at the Aug 31 gate (D6).

The three biggest findings:

1. **Documentation only**: The Phase 0 completion report contains a typo —
   "Calmodulin (apo) 6541" is an error. The benchmark actually contains
   **Calbindin D9k**, as correctly recorded in the authoritative
   `proteins/manifest.json`. BMRB 6541 is unrelated (SAIL calmodulin chemical
   shifts only). This is a report-only error; no pipeline impact. **Fix the
   completion-report table as a documentation correction.**
2. **Multiple BMRB IDs in the completion report are wrong** (bmr5852 is RNA,
   bmr4265 is dynein, bmr4355 is Mnt Repressor, bmr6541 is calmodulin). The
   manifest correctly uses "literature" for these proteins. The completion
   report's table is a cosmetic liability, not a data liability. Documentation
   fix only.
3. **Calbindin D9k apo/holo ambiguity**: PDB 3ICB is the Ca-loaded (holo) form,
   not apo as labeled. The BMRB "literature" references span both Kordel 1992
   (holo S2) and Akke 1993 (apo S2). This must be disambiguated *before* Phase 2
   runs begin — we need the MD ensemble state to match the S2 measurement state.
   Recommendation: lock to **apo** (use Akke 1993 S2 as reference, strip Ca2+
   from 3ICB before MD) since "apo" is what the manifest declares.

No RED flags emerged. Pin1 WW domain (2F21 Fip mutant), CI2 (2CI2 is 2.0 Å but
BMRB 51234 is the I57V mutant — reference-state mismatch with WT 2CI2), and the
T4L BMRB source (Mulder 2001 Biochemistry 40:4458 needs verification as WT vs.
cavity mutant) are all YELLOW: reference-state fidelity issues that need
careful handling in Phase 2 analysis, not protein drops.

Overall T5 status: **MET (12/12)**, zero margin confirmed as expected.

## Documentation Discrepancy: Calbindin D9k vs. Calmodulin

**Question asked:** The Phase 0 completion report's BMRB verification table
(lines 85-100) lists "Calmodulin (apo) 6541, High quality" but the manifest
and task-003 output show Calbindin D9k with literature references.

**Answer:** The benchmark contains **Calbindin D9k**, not Calmodulin. The
completion report's table is wrong in two ways:

1. **Protein name**: The 75-residue protein in the benchmark is Calbindin D9k
   (PDB 3ICB), not Calmodulin (which is 148 residues). This is unambiguous
   from `manifest.json` lines 155-173 and `task-003-bmrb-verification.md`
   line 29. See also the completion report's own item 8.2 (line 253) listing
   Calbindin D9k in the 14-protein set.

2. **BMRB ID**: BMRB 6541 is **Stereo-Array Isotope Labeled (SAIL) Calmodulin
   chemical shifts only** (591 13C, 158 15N, 751 1H shifts at 800 MHz, 310 K,
   pH 6.5; deposited by Kainosho lab). **No S2 data.** BMRB 6541 contains
   zero relaxation parameters and cannot be used as an S2 reference.

The completion report's table appears to have been generated with several
copy-paste errors:
- "Calmodulin 6541" should be "Calbindin D9k literature" (Kordel 1992 / Akke 1993)
- "WW domain 5852" should be "WW domain literature" (Seewald 2007). Actual BMRB
  5852 is an RNA ribozyme substrate.
- "CspA 6342" should be "CspA literature" (Feng 1998).
- "T4 Lysozyme 4355" should be "T4 Lysozyme literature" (Mulder 2001). Actual
  BMRB 4355 is Mnt Repressor from P22.
- "HPr 4265" should be "HPr literature" (van Nuland 1995). Actual BMRB 4265 is
  Dynein light chain 1.
- "Ubiquitin 6457" should be "Ubiquitin 6470". Actual BMRB 6470 is confirmed
  valid (Tjandra 1995 / 126 S2 values).
- "BPTI 4968" is a plausible BMRB reference but was for the dropped protein.
- "HEWL 5531" is a plausible BMRB reference but was for the dropped protein.

**Action required (documentation only, no data impact):** Correct the completion
report to match the manifest. The manifest is the authoritative source;
downstream pipelines consume the manifest, not the completion report.

**Pipeline impact**: ZERO. `proteins/manifest.json` is correct and is what
scripts read.
