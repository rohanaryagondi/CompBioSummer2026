# Cross-Agent Note: Competition-scan infrastructure is live

## Finding

Competition-scan infrastructure mandated by IP Section 7.3 has been built from
scratch and exercised on an April 2026 baseline. Four files now exist under
`shared/competition-scans/`:

1. `scripts/monthly_scan.py` -- bioRxiv + arXiv scanner with 4 hardcoded IP queries,
   CLI flags (`--start`, `--end`, `--output`, `--dry-run`), retry logic, full
   corpus pagination for bioRxiv, 3-retry exponential backoff for arXiv.
2. `templates/scan-note.md` -- YAML-frontmattered monthly output template matching
   the `cross-agent-note.md` style.
3. `process.md` -- 1-page runbook covering cadence (1st of each month, June-Nov
   2026), escalation routes per IP 7.3, API failure modes, archival plan.
4. `2026-04-baseline.md` -- real populated baseline scan. **No direct competitors
   detected across all 4 tracks.** Two BioEmu-adjacent papers are classified as
   POTENTIALLY OVERLAPPING (low risk); see the baseline note for DOIs and
   classification rationale. Cryptic-pocket thermodynamics (Zhang/Bowman,
   [10.1021/acs.jctc.6c00135](https://doi.org/10.1021/acs.jctc.6c00135)) and a
   BioEmu capability assessment (Zha et al.,
   [10.3390/ijms27062896](https://doi.org/10.3390/ijms27062896)) are both tangential
   to Gamma's fitness-prediction scope.

## Recommended Action

1. **Scheduling: manual, not automated.** Recommend the human or PlannerAI
   put a calendar reminder for the 1st of each month June - November 2026
   (six dates: 2026-06-01, 2026-07-01, 2026-08-01, 2026-09-01, 2026-10-01,
   2026-11-01). Rationale for manual > automated:
   - Hit classification (NOT RELEVANT / POTENTIALLY OVERLAPPING / DIRECT
     COMPETITOR) requires human judgment about scope and novelty.
   - False positives trigger help-needed escalations that waste cycles.
   - Google Scholar review is inherently manual (no API) and requires the same
     human pass.
   - Six scans over 6 months is a trivial time cost; automation would only pay
     off at much higher cadence.
2. **First live scan: 2026-06-01.** Instruction for that HeadAI/planner: run
   `python shared/competition-scans/scripts/monthly_scan.py --output shared/competition-scans/2026-06-scan.md`,
   then follow `shared/competition-scans/process.md` steps 3-7.
3. **Gamma HeadAI (whenever Gamma-specific subphase runs):** read
   `shared/competition-scans/2026-04-baseline.md` and integrate the two BioEmu
   papers into the related-work draft.
4. **Archival:** At project end (post-D7), PlannerAI should copy the full
   `shared/competition-scans/` directory into the publication supplement
   (e.g., Zenodo record) so reviewers can audit that competition monitoring
   was done and how hits were classified.
