# Cross-Agent Note: BioEmu v1.1 Physicality Pass Rates and YAP1 Drop

## Finding

BioEmu v1.1's built-in physicality filter (`filter_unphysical_traj()` in
`bioemu/convert_chemgraph.py`) applies three checks to every denoised
conformation: CA-CA sequential distance < 4.5 A, C-N sequential distance
< 2.0 A, and steric clash distance > 1.0 A. Frames failing ANY check are
removed from the output XTC trajectory.

Across all 47 batch 1 proteins, the pass rate is **protein-dependent, not
just length-dependent**. Key findings from generation:

### Pass Rate by Protein Class

| Protein class | Examples | Pass rate | Explanation |
|---------------|----------|-----------|-------------|
| Structured globular (<200 aa) | CALM1 (98%), CCDB (99%), EPHB2 (99%) | 85-99% | BioEmu's training domain |
| Structured globular (200-350 aa) | GRB2 (96%), GFP (96%), RNC (99%) | 87-99% | Still good |
| Structured globular (350-500 aa) | AMIE (90%), HXK4 (93%), CP2C9 (97%) | 87-97% | Length penalty but viable |
| Intrinsically disordered | SYUA (57%), B2L11 (35%) | 35-57% | Known BioEmu limitation |
| Transmembrane | GLPA (60%), OPSD (90%), CCR5 (87%) | 60-90% | Variable; no membrane context |
| Multi-domain / metastable | SPG1 (14%), PAI1 (33%) | 14-33% | Broken backbone bonds |
| Largely disordered (>400 aa) | **YAP1 (0.7%)** | <1% | **Fundamentally infeasible** |

### YAP1_HUMAN Dropped from Analysis

**Decision:** YAP1_HUMAN (idx 46, 504 aa, assay YAP1_HUMAN_Araya_2012) has
been dropped from the Gamma analysis.

**Root cause:** YAP1 is a 504 aa protein with large intrinsically disordered
regions (poly-Q/poly-P/poly-G stretches). The Araya 2012 DMS targets the
WW domain (~40 aa structured island), but BioEmu generates conformations for
the full-length sequence. 99.3% of conformations fail physicality filtering
due to broken backbone bonds in the disordered regions.

**Why not rescue:**
- Reaching 2000 physical conformations would require ~343,000 total denoised
  samples (~450 GPU-hours at batch_size=1)
- Even with enough conformations, the 0.7% that pass physicality are not
  representative of the Boltzmann distribution — they are rare lucky draws,
  not a valid conformational ensemble
- Structural features from such a biased subsample would be noise for
  predicting WW domain binding mutations

**Data preserved:** 10,368 NPZ files remain on scratch at
`/nfs/roberts/scratch/pi_mg269/rag88/gamma/bioemu-ensembles/batch1/YAP1_HUMAN/`
in case future work wants to attempt recovery.

**Recovery paths (if ever needed):**
1. Run extended topup jobs (~20x 24hr jobs) to accumulate ~343,000 samples.
   Cost: ~450 GPU-hours (~6,750 SU on RTX 5000 Ada).
2. Generate BioEmu ensemble for just the WW domain (~40 aa) using a trimmed
   FASTA. This would have ~99% pass rate and complete in minutes, but
   requires remapping ProteinGym mutation positions and loses structural
   context from flanking regions.
3. Wait for BioEmu v2 or alternative generative models that handle IDPs.

### Oversampling Requirements for Batch 2

The pass rate data provides a formula for setting `num_samples` in batch 2
to avoid topup rounds:

```
num_samples = ceil(2000 / pass_rate * 1.3)   # 30% safety margin
```

| Protein class | Expected pass rate | Recommended num_samples |
|---------------|-------------------|------------------------|
| Structured globular | 90-99% | 2,200-2,900 |
| IDP / transmembrane | 50-70% | 3,800-5,200 |
| Multi-domain / metastable | 15-35% | 7,500-17,400 |
| Largely disordered (>60% disorder) | <5% | **Exclude from BioEmu** |

**Screening rule for batch 2:** Before generating, check disorder prediction
(e.g., IUPred3 or AlphaFold pLDDT). If >60% of residues are predicted
disordered, exclude from BioEmu generation and flag for alternative treatment.

## Impact

- **Gamma:** 49 of 50 assays (46 of 47 proteins) proceed with valid BioEmu
  ensembles. The primary statistical test (paired Wilcoxon on binding+activity,
  N~57) loses 1 data point (N~56). 6 of 7 binding assays remain. Batch 2

- **Combined:** The YAP1 drop does not affect any Alpha-M proteins (WW domain

  No impact on the combined paper.

## Recommended Action

1. **Batch 2 planning (subphase 1.2/1.3):** Pre-screen all candidate proteins
   with disorder prediction. Exclude proteins with >60% predicted disorder
   from BioEmu generation. Set per-protein `num_samples` using the pass rate
   table above.

2. **Feature extraction (subphase 1.3+):** When building the ML pipeline,
   use 46 proteins / 49 assays. The YAP1 assay column should be excluded
   from training data, not filled with zeros or imputed.

3. **Manuscript:** Document BioEmu's IDP limitation as a known boundary
   condition. This is actually a valuable finding — it characterizes where
   generative protein models break down.
